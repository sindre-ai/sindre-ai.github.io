The manifesto

When humans and AI
work together

Sebastian Krumhausen & Magnus Nøddegaard · sindre.ai

For fifty years, the tech industry's most urgent problem was the same: how do you protect expensive engineering time from being wasted?

Every methodology we built was an answer to that question. Waterfall gave us sequential phases so nothing got built before it was specified. Agile gave us sprints so scope couldn't sprawl. Scrum gave us ceremonies — standups, retrospectives, planning poker — so the team stayed aligned without constant interruption. SAFe scaled it across programs and portfolios. OKRs connected company strategy to quarterly execution so engineers were never building the wrong thing.

And someone had to manage all of it. Product managers became the guardian of the process — the translator between customer and engineer, the connective tissue holding the system together. Decisions lived in meetings. Context lived in people's heads. Signals arrived in five different tools and had to be manually assembled into something an engineer could act on.

The human was the integration layer. Because nothing else could be.

We got good at it. So good that we forgot: the process was never the product. It was the cost of coordination. And we were optimising the cost, not the outcome.

And with it, the entire operating model

What took a sprint now takes an afternoon. What required three engineers now requires one with an agent. The cost of implementation has dropped by an order of magnitude — and it's still falling.

The scarcity that justified the entire operating model — the reason every request needed a ticket, every ticket needed a priority, every priority needed a meeting — is gone.

And now the question you couldn't ask before is unavoidable:

If an agent can write the PRD, generate the roadmap, and triage the backlog — were those ever really product decisions?

They were coordination overhead dressed up as product work. The process was necessary. It was never the job.

The first response

The instinct is to use AI to do the old job faster. Generate the roadmap. Write the PRD. Summarise the standup.

It feels productive. It is a trap.

Accelerating the overhead isn't escaping it. The teams treating AI as a productivity layer on top of the existing model are still running the same operating model — just at higher velocity toward the same destination.

The opportunity was never speed. It's replacement.

What the job actually is

Strip away the process and what's left is the work people were always supposed to do.

Judgment. Which market? Which bet? Which problem is worth solving right now? These are taste calls, not logic problems. No amount of data resolves them — someone has to make the call and own the outcome.

Creative leaps. The insight that connects two unrelated signals into a hypothesis nobody's tested. The pattern that lives in a conversation from last week, not in the data yet. Agents can surface signals. They can't make the jump.

Relationships. The meeting. The handshake. The "I trust you" that closes the deal. Agents can draft the outreach, research the prospect, time the follow-up. Humans close it.

Craft. The decision that makes something feel right. The quality bar that separates good enough from genuinely good.

This is the job. It was always the job. It just got buried under everything else.

Maskin

Making this work requires more than better prompts. It requires a shared environment — where agents and humans see the same data, share context, and hand off to each other without a human acting as the integration layer. Where agents are not tools you query and put down, but digital colleagues with persistent memory, defined roles, and ongoing responsibilities.

Maskin is that environment. An open-source agent operating system — a framework for orchestrating humans and AI agents over a shared, extensible knowledge graph.

Actors. Humans and agents are first-class citizens with the same API surface. Both can create objects, update state, traverse relationships, and trigger workflows. Agents carry a system prompt, persistent memory, configurable tools, and their own LLM. Everything speaks MCP. Bring your own model.

Objects and the scientific method. Three primitives mirror the oldest rigorous loop for operating under uncertainty: observe, hypothesise, experiment.

  • Insights capture signals.
  • Bets turn them into testable hypotheses with defined investment and success criteria.
  • Tasks make them happen. The middle step — forming the hypothesis before jumping to execution — is what most tooling skips. It's also why teams build things nobody wanted.

The graph. Everything connects. An insight informs a bet. A bet breaks into tasks. A task blocks another. Agents navigate this graph the way you'd navigate a codebase — by following relationships, not re-reading every document from scratch. Extensions add domain-specific types on top. Triggers fire agents on schedules or state changes. Workspaces scope it all.

Build the machine that works for you

The same primitives compose differently depending on the team.

A product team.

A customer signal arrives. The Insight Scout Agent catches it, enriches it, links it to a cluster of related signals — the fourth one this week pointing the same direction. The Bet Shepherd sees the cluster and proposes an experiment: two weeks, a clear success metric, a scoped investment.

The product manager gets a notification. Reviews the proposal. Accepts it. One decision.

The Bet Decomposer breaks it into tasks with dependencies mapped. The Task Nagger monitors progress. The Daily Briefing closes the loop each morning — what moved, what's stuck, what needs a call today.

A growth team.

No signal triggered this. The Pipeline Scout has been running weekly analysis on segment performance and spots the pattern before anyone asks: one ICP segment is converting at 3x the average, but outreach volume is flat. It surfaces the opportunity.

The Growth Strategist agent proposes a bet: a two-week campaign to double outreach to that segment. The SDR Agent drafts personalised messages for each contact — researched, unique, queued for human review before anything sends. The Content Creator turns the initiative into supporting assets. The Curator scores each piece against audience data and surfaces the strongest version.

The revenue lead reviews, approves the outreach, and adjusts one message. The rest runs.

In both cases: agents carry the process. Humans carry the product.

Why open source

An agent operating system only works if you trust it. How decisions flow, how data moves, how agents behave — that should be visible, not hidden behind an API key.

Open source means you can inspect every layer, run it on your own infrastructure, bring your own models, and extend it for your domain.

The best teams will be the ones where the process runs itself — so the humans can do the work only humans can do.
